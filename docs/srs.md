# 基于用户行为感知的动态专注管理系统 (FlowState) 需求规格说明书 (SRS)

## 1. 系统概述
FlowState 是一款面向数字媒体技术专业学生的动态专注管理系统。系统通过冷启动行为画像和用户状态容错策略，解决传统番茄钟显式申报繁琐、任务分配僵化的问题。

## 2. 功能需求规格

### 2.1 冷启动用户画像模块 (FR-01)
- **输入**：用户首次登录时的职业分类选项、感兴趣领域文本。
- **处理**：系统拦截初始请求，读取本地静态 JSON 种子数据集（包含低、中、高精力任务各 1 条）。
- **输出**：将 3 条硬编码的推荐任务自动持久化写入 LocalStorage，并在视图层打上分类标签。

### 2.2 动态专注匹配与容错模块 (FR-02)
- **输入**：用户选择的当前精力状态（HIGH / MEDIUM / LOW / UNCERTAIN）。
- **处理逻辑**：
  1. 若输入为 HIGH/MEDIUM/LOW，系统检索对应 `energy_level` 且 `status == 'PENDING'` 的最旧任务。
  2. 若输入为 UNCERTAIN（不确定），系统触发启发式降级策略，在本地的任务队列中通过随机函数（Math.random）抽取一条 MEDIUM 或 LOW 精力的任务执行。
  3. 若检索为空，抛出高可用性异常拦截弹窗。
- **输出**：激活倒计时状态机，切换界面为低饱和度专注视觉反馈层。

### 2.3 灵感池管理模块 (FR-03)
- **输入**：非结构化灵感文本（如“想做一个小网页”）。
- **处理**：系统不启用高耗能的 NLP 分类器，直接采用硬编码兜底配置，将其转换为：`energy_level = 'MEDIUM'`, `duration_minutes = 25`, `source = 'INSPIRATION_POOL'` 的规范化数据结构。
- **输出**：向任务池数组追加该对象。允许用户在列表层通过交互事件手动重写其属性。

## 3. 非功能需求规格
- **易用性**：用户从进入应用到开启专注状态，核心交互路径长度 $\le 2$ 步。
- **性能**：倒计时状态机的视觉线性插值转换帧率 $\ge 60\text{fps}$。
