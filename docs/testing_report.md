# 基于用户行为感知的动态专注管理系统 (FlowState) 测试计划与报告

## 1. 测试计划与方法概述
本测试阶段采用黑盒测试（Black-box Testing）方法，主要针对核心业务逻辑进行等价类划分与边界值分析。
- **测试环境**：Google Chrome (v120.0) / Edge 浏览器
- **测试框架**：基于 Vue 3 状态驱动的断言测试

## 2. 核心功能测试用例与执行结果

### 2.1 冷启动拦截测试 (TC-01)
- **测试场景**：用户首次打开系统（LocalStorage 无数据）。
- **预期结果**：系统成功拦截主界面，弹出职业与兴趣询问遮罩层；提交后本地自动写入 3 条预置种子任务。
- **实际结果**：遮罩层正常阻断交互，点击提交后 `flowstate_tasks` 成功初始化 3 条带有推荐标签的数据。
- **状态**：通过 (Passed)

### 2.2 “不确定 (UNCERTAIN)”精力状态降级匹配测试 (TC-02)
- **测试场景**：用户任务池中同时存在高、中、低精力任务，用户选择“不确定”状态并点击开始专注。
- **预期结果**：系统应自动过滤掉高精力任务，仅在 pending 的中、低精力任务中进行随机抽取执行。
- **实际结果**：系统触发降级路由，成功规避高精力任务，提示信息显示“已为你自动匹配适中精力的任务”。
- **状态**：通过 (Passed)

### 2.3 灵感池任务特征转化测试 (TC-03)
- **测试场景**：在灵感池中输入非结构化文本“看一部数媒纪录片”并提交。
- **预期结果**：任务成功追加到列表，无日期和时长约束限制，系统默认赋予 MEDIUM 精力与 25 分钟时长，并带有“来自你的灵感池”高亮标签。
- **实际结果**：任务属性正确转化，视图层渲染出对应高亮标签，允许二次修改。
- **状态**：通过 (Passed)

### 2.4 专注结束视觉反馈线性插值测试 (TC-04)
- **测试场景**：倒计时由 00:01 递减至 00:00。
- **预期结果**：静止的灰色背景在 2 秒内平滑转换为高饱和度极光渐变色。
- **实际结果**：CSS `transition` 属性在 Vue 状态机变更时正确触发，色域平滑过渡无闪烁。
- **状态**：通过 (Passed)

## 3. 测试结论
本系统经过等价类边界测试，三大创新核心功能逻辑无死锁，本地模拟持久化层 I/O 异常处理机制完备，符合软件工程期末交付质量标准。
